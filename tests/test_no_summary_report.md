# 带摘要 vs 仅标题 LLM 分析效果对比报告

> 测试时间: 2026-03-20 00:53 ~ 00:56
> 测试模型: 豆包 doubao-seed-2.0-lite

## 背景

当前分析流程：头条搜索 → 取 `title + summary[:200]` → 格式化为编号文本 → 喂给 LLM 分析

用户提出：头条搜索结果本身就是摘要/摘要片段，再把 summary 喂给 LLM 是否多余？

## 测试方案

对同一批搜索结果，分别用两种格式调用 LLM：

| 模式 | 格式 | 说明 |
|------|------|------|
| 带摘要（现有） | `{i}. [{date}] {title}\n{summary}` | title + 搜索页摘要（≤200字符） |
| 仅标题（实验） | `{i}. [{date}] {title}` | 只保留标题 |

## 测试结果

### 1. 个股分析：贵州茅台 (600519)

| 指标 | 带摘要 | 仅标题 |
|------|--------|--------|
| Prompt 长度 | 5643 字符 | 2635 字符 |
| **节省** | - | **53.3%** |
| LLM 耗时 | 32.7s | 43.7s |
| 总分 | 68 | 83 |
| sentiment | 中性 | 乐观 |

**子维度对比：**

| 维度 | 带摘要 | 仅标题 | 差异 |
|------|--------|--------|------|
| growth | 45 | 80 | +35 |
| profitability | 80 | 90 | +10 |
| competitive | 95 | 95 | 0 |
| risk_level | 80 | 85 | +5 |
| catalyst | 40 | 60 | +20 |

**分析质量对比：**

- **带摘要**：提到了更具体的数据（Q3营收390.64亿、同比+0.56%），分析更细致，识别了"行业库存周转天数900天"等风险
- **仅标题**：分析偏正面/乐观，缺少负面细节，但覆盖面更广（提到了产能达5.6万吨等长期规划）
- **关键差异**：带摘要能提取摘要中的具体数字（如Q3增速0.48%），仅标题只能从标题推断（"利润微增"）

### 2. 国际形势分析

| 指标 | 带摘要 | 仅标题 |
|------|--------|--------|
| Prompt 长度 | 4408 字符 | 1762 字符 |
| **节省** | - | **60.0%** |
| LLM 耗时 | 17.8s | 16.8s |
| 总分 | 40 | 35 |
| sentiment | 利空 | 利空 |

**分析质量对比：**

- **两者结论一致**：sentiment 相同（利空），核心判断一致（美联储按兵不动、中东冲突升级、原油破110美元）
- **带摘要**：多提到"黄金与原油罕见背离"、"北向资金流入意愿"等细节
- **仅标题**：多提到"中美关系边际缓和"、"离岸人民币贬值"，信息来源更广

## 结论

### 信息量 vs Token 效率

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| **国际/国内形势** | ✅ 仅标题即可 | 宏观分析依赖趋势判断，标题已包含关键信号，两者结论几乎相同 |
| **个股分析** | ⚠️ 带摘要更好 | 个股需要具体数据（营收、利润、增速），摘要中常含关键数字 |
| **行业/板块分析** | ✅ 仅标题可行 | 类似宏观分析，标题通常能传达核心信息 |

### 核心发现

1. **Token 节省显著**：去掉摘要可节省 53%~60% 的 prompt 长度
2. **宏观分析影响小**：国际形势两种方式结论高度一致（sentiment 相同，score 差 5 分）
3. **个股分析有差异**：带摘要能捕获更具体的财务数据，分析更精准
4. **仅标题倾向乐观**：缺少细节时 LLM 倾向给出更正面的判断（茅台从 68→83）
5. **头条搜索摘要本身质量参差**：部分摘要是有价值的新闻开头，部分只是来源和评论数

### 建议方案

**折中方案**：对国际/国内/行业/板块分析去掉摘要（仅标题），个股分析保留摘要。这样可以在保持个股分析精度的同时，大幅节省宏观分析的 token 消耗。
